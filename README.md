## 🎯 核心原则

### 1. 安全隔离
- 所有项目文件仅在 `/Users/bingo/openclaw_all` 目录内
- 不修改用户其他重要文件
- 敏感操作需要人工确认

### 2. 任务审核
- 每个任务完成后需要审核确认
- 日志完整记录，可追溯
- 支持回滚和修正

### 3. 可扩展性
- 模块化设计，方便添加新 Agent
- 统一的通信协议
- 松耦合架构

## 🚀 快速开始

### 前置条件
- Node.js >= 18
- OpenClaw 环境已配置
- 消息平台集成（可选）

### 添加新 Agent

1. 在 `agents/` 下创建新目录
2. 定义 Agent 配置文件
3. 注册到任务分发层
4. 测试工作流

## 📊 状态追踪

所有 Agent 使用 `STATE.yaml` 模式追踪任务状态：

```yaml
agent: code-dev
status: idle  # idle | running | completed | error
current_task: null
last_update: 2026-03-01T12:00:00Z
logs:
  - timestamp: 2026-03-01T12:00:00Z
    level: info
    message: Agent initialized
```
