## 二、Agent 角色设计

### 2.1 基础形态

**设计理念：** 半透明数据流构成的"量子态"小人

```
        ╭─────────────╮
        │   头部      │  ← 发光光环（状态色）
        │  (头盔状)   │
        ╰──────┬──────╯
               │
        ╭──────┴──────╮
        │   身体      │  ← 半透明，内部数据流可见
        │ (数据流构成) │
        ╰──────┬──────╯
               │
        ╭──────┴──────╮
        │   底座      │  ← 悬浮平台，部门图标
        ╰─────────────╯
```

**高度比例：** 头:身:底座 = 1:2:0.5

---

### 2.2 部门分类与视觉标识

| 部门 | 主色 | 图标 | 特征 |
|------|------|------|------|
| **研发部** | #58A6FF 蓝 | 💻 | 头部有代码流环绕 |
| **内容部** | #D2A8FF 紫 | ✍️ | 身体有文字粒子飘散 |
| **调研部** | #3FB950 绿 | 🔍 | 头部有雷达扫描效果 |
| **数据部** | #F85149 红 | 📊 | 身体有图表投影 |
| **客服部** | #FFB75E 橙 | 💬 | 周围有对话气泡 |
| **投资部** | #79C0FF 青 | 📈 | 底座有 K 线波动 |
| **营销部** | #FF7B72 粉 | 📣 | 周围有喇叭波纹 |
| **运营部** | #A5D6FF 浅蓝 | ⚙️ | 身体有齿轮转动 |

---

### 2.3 状态表现

| 状态 | 光环颜色 | 动效 | 音效（可选） |
|------|----------|------|-------------|
| 🟢 **空闲** | 绿色呼吸 | 缓慢上下浮动（0.5Hz） | 轻微嗡鸣 |
| 🟡 **忙碌** | 黄色脉冲 | 快速闪烁（2Hz）+ 数据流加速 | 键盘敲击感 |
| 🔴 **异常** | 红色闪烁 | 剧烈抖动 + 粒子散开 | 警报声 |
| ⚪ **离线** | 灰色静止 | 半透明渐隐 | 无 |
| ⭐ **高效** | 金色光环 | 周围有星星粒子 | 升级音效 |

## 三、交互动效设计

### 3.1 悬停效果

```
用户鼠标悬停在 Agent 上
    ↓
1. Agent 放大 1.2 倍（0.2s ease-out）
2. 光环亮度 +50%
3. 显示信息卡片（从底部滑入）
4. 底座部门图标高亮
```

**信息卡片内容：**
```
┌─────────────────────┐
│  🤖 调研部 #3       │
│  状态：🟡 忙碌      │
│  当前任务：扫描 AI 新闻│
│  进度：████░░ 67%   │
│  耗时：2 分 34 秒    │
└─────────────────────┘
```

---

### 3.2 任务执行动画

```
任务开始执行
    ↓
1. Agent 光环变为黄色脉冲
2. 从 Agent 发射粒子流到任务卡片
3. 粒子流中显示任务图标
4. 任务卡片进度条随粒子流动增长
    ↓
任务完成
    ↓
1. 光环变绿，闪烁 3 次
2. 烟花粒子效果（0.5s）
3. 显示"✓ 完成"浮层
```

---

### 3.3 部门视图切换

```
用户点击"研发部"筛选
    ↓
1. 非研发部 Agent 淡出（opacity 0.3）
2. 研发部 Agent 向中心聚拢
3. 相机轻微推进（zoom 1.1）
4. 背景网格高亮对应区域
```

## 四、3D 技术实现方案

### 4.1 模型格式

- **格式：** glTF 2.0（.glb 二进制）
- **面数：** 每个 Agent < 5,000 三角面（移动端友好）
- **纹理：** 1024×1024 PNG，透明通道
- **动画：** 骨骼动画（空闲/忙碌/完成 3 套）

---

### 4.2 渲染优化

| 技术 | 说明 | 预期效果 |
|------|------|----------|
| 实例化渲染 | 相同 Agent 共用几何体 | 100+ Agent 仍 60fps |
| LOD | 远距离简化模型 | 减少 GPU 负载 |
| 遮挡剔除 | 不渲染被遮挡的 Agent | 提升性能 |
| 延迟渲染 | 多光源场景优化 | 支持复杂光照 |

---

### 4.3 MVP 降级方案

**如果 3D 开发时间不足：**

| 方案 | 描述 | 开发时间 |
|------|------|----------|
| **2D 精灵** | 预渲染的 PNG 序列帧 | 2 天 |
| **Lottie 动画** | JSON 格式矢量动画 | 3 天 |
| **CSS 动效** | 纯 CSS 实现呼吸/脉冲 | 1 天 |
| **Three.js 基础几何** | 简单几何体 + 发光材质 | 3 天 |

**推荐：** MVP 用 CSS 动效 + 2D 图标，Phase 2 替换为 3D 模型

## 七、MVP 执行建议

### 推荐方案：**2D 优先，3D 渐进**

**Week 1-2（MVP）：**
- 用 CSS 动效 + 2D 图标实现 Agent 卡片
- 状态用颜色 + 简单动画（呼吸/脉冲）
- 快速上线验证核心功能

**Week 3-4（优化）：**
- 替换为 Three.js 基础几何体（球体 + 圆柱）
- 添加发光材质和简单粒子
- 保持 60fps 性能

**Phase 2（完整版）：**
- 导入专业 3D 模型
- 完整粒子系统和音效
- AR 视图（移动端）

## 八、设计原则

### 8.1 性能优先

- 首屏加载 < 2s
- 60fps 流畅动画
- 支持 100+ Agent 同屏

### 8.2 渐进增强

- 低端设备自动降级（2D）
- 高端设备开启 3D+ 粒子
- 用户可手动切换

### 8.3 一致性

- 所有状态变化有动效过渡
- 颜色语义统一（绿=好，红=坏）
- 图标风格一致（线性/填充）
